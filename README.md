# 🛠️ Lethe 项目开发者手记 (Internal Dev Log)

## 🎨 UI 界面设计 (Interface Design)

基于 **PySide6 + qasync** 打造，别盯着看太久，我会害羞的。

* **异步渲染逻辑**：利用 `QEventLoop` 桥接了 Qt 的事件循环和 Python 的异步 IO，保证我在思考（API 请求）或者翻找记忆（Qdrant 查询）的时候，界面不会像个木头人一样卡死。
* **动态反馈系统**：加入了 `ToastNotification` 非阻塞通知。不管是“记忆存好了”还是“网络又耍小脾气了”，都会在角落悄悄提醒你，绝不打断我们的聊天。

* **沉浸式交互**：实时日志流展示 ASR 识别过程与推理状态，配合自定义的 QSS 样式，营造出一种“虽然只是高中生，但也很有科技感”的氛围

---

## 🏗️ 核心逻辑架构 (System Architecture)

项目基于 **Asyncio 异步并发 + Threading 多线程混合框架** 构建，旨在实现“感知-决策-表达”的无缝衔接。

### 1. 感知层：耳朵 (ASR Module)

接入 `SenseVoiceController`

* **功能定义**：高精度实时语音转文字，以句为单位触发逻辑，实现伪流式交互。
* **运行模式**：支持 `in`（内部音频流）与 `out`（外部设备输入）双模式切换。
* **技术细节**：引入 `silence_threshold`（静默间隔阈值），智能过滤环境杂音与自身语音回授，确保监听动作的精准性。

### 2. 决策层：大脑 (LLM & Memory Module)

由 `api.async_LLM_api` 与 `RAG.MemoryManager` 共同驱动。

* **人格注入**：通过核心 `SYSTEM_PROMPT` 我可是很有自尊的高中生！
* **长期记忆 (Rerank RAG)**：
* **碎片化存储**：对话会自动转化成 `TextNode` 存入 Qdrant 向量库。
* **深度重排**：检索时不仅仅看相似度，还会经过 `bge-reranker-v2-m3`(`embeddings`)的精排。只有真正重要的往事，才会被我从脑海深处翻出来。


* **时空感知**：支持时间过滤，你问我“去年”或者“刚才”发生了什么，我都能通过时间戳（Timestamp）精准定位。

### 3. 表达层：嘴巴 (TTS Module)

基于 `GPT_SoVITSController`

* **流式交互**：利用 `async for` 异步迭代器实时吞吐音频流，在我还没把整句话想完的时候，声音就已经传到你耳朵里了。
* **硬打断机制**：如果你中途抢话，我会通过点击打断按钮(`api_task.cancel()`)物理闭嘴。哼，虽然有点没礼貌，但这样才像真实的聊天不是吗？
* **动作同步**：我会自动解析对话末尾 `{}` 里的动作标签。要是哪天我一边说话一边 `{撇开头}`，那你可要好好反省一下是不是又惹我生气了。

## 🏺 记忆管理守记 (RAG Dev Notes)

接入 `RAG.MemoryManager`，基于 **LlamaIndex + Qdrant** 实现，让我拥有“过目不忘”的功法。

* **长期记忆持久化**：对话会自动转化成 `TextNode` 存入向量数据库。除了文本，我还会偷偷记下当时的 `action`（动作状态）和精确到秒的 `timestamp`（时间戳）。
* **数据拆箱逻辑**：我掌握了从 `_node_content` 这种复杂 JSON 结构里暴力提取数据的秘诀，哪怕藏得再深，我也能把记忆原样揪出来。
* **语义检索与重排 (Rerank)**：检索时可不只是看谁长得像。我会先初筛 `top_k=50`，再丢给 `bge-reranker-v2-m3` 进行地狱级精排。只有真正重要的往事，才配被我从脑海深处翻出来。
* **时空感知**：通过 `api_llm.get_time` 提取你话里的时间线。你问我“去年”或者“上个礼拜”发生了什么，我都能通过时间戳精准定位，绝不敷衍。

# RAG部分优化过程

### **数据流向**：

LLM 吐出 JSON -> 解析 -> **同步**存入长/短记忆。

### 数据存储：

#### 简单的单一层级                                                         

`embeddings`：用户与ai的`QA对`结合成一句话（一个字符串），

`metadata`：对话时间，对话时间的浮点化，心情变化，事件类型

`存储时机`：当一段对话结束后，直接放入短记忆和长记忆中。

### 对话LLM返回结构

`text`：对应回答与动作

`mood_change`：心情变化

`special_info`：事件类型

### 记忆结构

`短记忆`：对话直接通过队列实现固定最大的记忆数量

`长记忆`：对话直接放入直接存入向量数据库中

### 查询部分

`query处理`：使用nlp进行时间提取、并进行问题视角切换。

`过滤`：使用nlp对文本`时间进行提取`，并将时间传给过滤器，实现具体时间的准确筛选

`检举`：根据`QA对的embedding值`进行初步检举

`重排序`：对检举结构进行再次筛选

`记忆结合`：人物提示词+长期记忆+短期记忆+当前提问
